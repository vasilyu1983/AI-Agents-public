# AIOps Patterns (2026)

AI-assisted operations for infrastructure, cost, incident management, and platform engineering.

> AI recommendations need human validation before action. Automated deletions require approval workflows. Self-healing actions need human-defined policies and audit trails.

---

## Self-Healing Systems

- **Anomaly detection** — Predict failures before they happen using ML on metrics/logs
- **Automated remediation** — Trigger rollbacks or config changes via predefined runbooks
- **Intelligent test selection** — Risk-based change scoring in CI/CD pipelines
- **Causal graph analysis** — Instant root cause identification across service dependencies

## Automated Operations

- **Unused resource detection** — Surface idle/orphaned resources for cleanup
- **Right-sizing recommendations** — ML-driven compute/memory suggestions from utilization data
- **Alert summarization** — Correlate and deduplicate alerts (reduce noise by 90%+)
- **Runbook automation** — AI-suggested steps with human-approved execution

## AI-Assisted Analysis

- Cost trend prediction and anomaly detection
- Incident pattern identification across services
- Post-mortem theme extraction and recurring issue flagging
- Capacity planning predictions from historical growth

## Platform Engineering + AI

Platform teams increasingly embed AI capabilities directly into the platform:

- Multi-agent orchestration for code generation, security validation, deployment
- Intelligent defaults and guardrails that scale across teams
- AI-powered developer portals that suggest golden paths based on project context

## Bounded Claims

- AI recommendations need validation before action
- Automated deletions require approval workflow
- Cost predictions are estimates, not guarantees
- Runbook suggestions need SRE verification
- Self-healing actions should have human-defined policies and audit trails
