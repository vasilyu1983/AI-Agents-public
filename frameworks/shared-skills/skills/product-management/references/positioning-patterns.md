# Positioning Patterns  

*Operational guide for crafting, testing, and validating product positioning.*

This file contains ONLY:

- Templates  
- Checklists  
- Step-by-step workflows  
- Decision trees  
- No theory or stories  

---

# 1. 10-Step Positioning Process (Operationalized)

Follow in sequence. Each step must produce a tangible artifact.

### **Step 1 — Identify Best-Fit Customers**

Artifacts:

- [ ] List of most successful customers  
- [ ] Shared attributes (segment, behavior, needs)  
- [ ] Exclusions list (who is NOT the target)  

### **Step 2 — Positioning Team**

- [ ] PM  
- [ ] PMM  
- [ ] Eng Lead  
- [ ] Sales/CS rep with customer proximity  
- [ ] Decision authority identified  

### **Step 3 — Vocabulary Reset**

Deliverable:

- [ ] List of banned internal jargon  
- [ ] Customer-language replacements  
- [ ] Agreement on literal definitions  

### **Step 4 — Competitive Alternatives**

Template:
Alternative | Customer's “job” for it | Strengths | Weaknesses | Switching triggers

### **Step 5 — Unique Attributes**

Attributes must be:

- [ ] Provable  
- [ ] Relevant  
- [ ] Hard for competitors to match  
- [ ] Tied to customer outcomes  

### **Step 6 — Value Themes**

Convert attributes → customer value.

Template:
Attribute → What it lets users do → Tangible value → Why this matters

### **Step 7 — “Who Cares Most?” Segments**

Score 1–5:

- [ ] Pain severity  
- [ ] Problem frequency  
- [ ] Existing workaround cost  
- [ ] Tech readiness  
- [ ] Purchase authority  

### **Step 8 — Frame of Reference**

Choose *one*:

- [ ] New category  
- [ ] Market you elevate  
- [ ] Subcategory  
- [ ] Competitor comparison  
- [ ] Adjacent category  

Rules:

- [ ] The frame must help customers “get it” fast  
- [ ] Must reduce—not increase—explaining  

### **Step 9 — Add a Trend (Optional)**

Use only if:

- [ ] It strengthens your differentiation  
- [ ] Customers already care  
- [ ] Not a stretch  

### **Step 10 — Capture the Positioning**

Use the template in section 2.

# 2. Positioning Template

For: [Target segment]

Who: [Description of users + specific context]

Our product is a: [Category / frame of reference]

That helps them: [Key job / outcome]

By: [Value themes expressed in customer language]

Unlike: [Competitive alternatives]

Our product: [Differentiated attributes that matter]

Rules:

- Use real customer language only  
- Remove adjectives not tied to proof  
- Replace “fast, easy, simple” with evidence (numbers, examples)

# 3. Competitive Mapping Patterns

## 3.1 Competitive Alternatives Table

Alternative | Job it does | Strengths | Weaknesses | Why they choose it | Why they switch

## 3.2 Attribute Mapping Grid

Score 1–5:

- Relevance to customer  
- Competitor performance  
- Our performance  
- Differentiation strength  

## 3.3 Trap Avoidance Checklist

- [ ] Do NOT list features as differentiators  
- [ ] Do NOT list things competitors also claim  
- [ ] Do NOT list internals (“our ML pipeline…”)  
- [ ] Do NOT list future features  
- [ ] MUST tie every differentiator to customer value

# 4. Value Theme Patterns

Turn attributes into “Why it matters.”

### Pattern A — Functional Value

Attribute → Enables user to [action] → Saves time / reduces cost / increases output

### Pattern B — Emotional Value

Attribute → Reduces risk / increases confidence → Feel safer / feel in control

### Pattern C — Social Value

Attribute → Helps user align with team / customer expectations → Reputation or credibility boost

### Value Theme Checklist

- [ ] Themes cluster attributes into 2–4 groups  
- [ ] Each theme contains customer-language outcomes  
- [ ] Each theme has at least 1 proof point

# 5. Messaging Patterns (Operational)

### 5.1 Problem-First Pattern

Use when category is crowded.

Lead with problem
Explain cost of current workarounds
Reveal your new approach
Make differentiation obvious

### 5.2 Value-First Pattern

Use when value is fast and measurable.

Lead with core value
Evidence (metric)
How it works (simple)
Differentiators

### 5.3 Outcome Narrative Pattern

Use for sales, exec decks.

Before → After → How we get you there → Proof

# 6. Positioning Validation

## 6.1 Message Testing Checklist

Test with:

- [ ] Best-fit customers (5–8 interviews)  
- [ ] Prospects (5–10 interviews)  
- [ ] Sales & support teams  

Look for:

- [ ] “I get it” within 5–10 seconds  
- [ ] They can explain it back in their own words  
- [ ] They can identify who it’s for  
- [ ] They can identify why it's different  
- [ ] They know what to do next  

## 6.2 Fail Conditions

If any of these occur, positioning must be revised:

- [ ] Users confuse you with a competitor  
- [ ] Users misinterpret the category  
- [ ] Benefits feel generic  
- [ ] Proof doesn't land

# 7. Decision Trees

## 7.1 Is Positioning Ready?

Do customers understand what you do in <10 seconds?
├─ No → simplify category/frame
└─ Yes
↓
Do they understand why it matters to them?
├─ No → clarify value themes
└─ Yes
↓
Do they understand what makes you different?
├─ No → refine differentiators + proof
└─ Yes → Positioning is ready

---

## 7.2 Which Frame of Reference?

Is the category saturated?
├─ Yes → Choose subcategory or competitor comparison
└─ No
↓
Do users already understand similar products?
├─ Yes → Adopt known category (fast comprehension)
└─ No → Create new category (requires education)

---

# 8. Anti-Patterns

- AVOID: Leading with features  
- AVOID: Overclaiming (“AI-powered”, “all-in-one”, “simple”)  
- AVOID: Category creation without evidence  
- AVOID: Positioning that applies to everyone  
- AVOID: Lists of adjectives without proof  
- AVOID: Differentiators that optimize for internal preferences  
- AVOID: Positioning not tested with customers  

---

# 9. Definition of Done (Positioning)

A positioning statement is **ready** when:

- [ ] Captures 1 customer segment (not multiple)  
- [ ] Includes clear category/frame  
- [ ] Lists 2–4 value themes  
- [ ] Lists 2–3 differentiated attributes  
- [ ] Attributes → proof points  
- [ ] Validated with customers  
- [ ] Aligned with sales + product + leadership

# 10. AI Product Positioning (Jan 2026)

Special patterns for positioning AI-powered products.

## 10.1 The Killer Question

> "If everyone has AI, what makes YOUR AI different?"
> — April Dunford, 2026

When "We have AI" is table stakes, differentiation must go deeper.

## 10.2 AI Positioning Checklist

- [ ] Differentiation goes beyond "We have AI"
- [ ] Specific outcome the AI delivers is named
- [ ] Data advantage is documented (proprietary data, unique training)
- [ ] Speed/accuracy/cost advantage is quantified
- [ ] Human-AI workflow is clear (not black box)
- [ ] Limitations are stated (builds trust)
- [ ] Positioning tested with actual customers

## 10.3 AI Differentiation Patterns

### Pattern A — Data Moat

"Our AI is trained on [unique data source] that competitors cannot access."

Proof required:

- [ ] Data source named
- [ ] Exclusivity explained
- [ ] Outcome improvement quantified

### Pattern B — Workflow Integration

"Our AI fits into [specific workflow] so you don't change how you work."

Proof required:

- [ ] Integration points listed
- [ ] Time-to-value measured
- [ ] Comparison to standalone tools

### Pattern C — Accuracy Advantage

"Our AI achieves [X% accuracy] on [specific task] vs. [Y% industry average]."

Proof required:

- [ ] Benchmark methodology transparent
- [ ] Test set described
- [ ] Third-party validation if possible

### Pattern D — Cost Efficiency

"Our AI delivers [outcome] at [X% lower cost] than [alternative]."

Proof required:

- [ ] TCO comparison
- [ ] Hidden costs included (training, maintenance)
- [ ] ROI timeline

## 10.4 AI Positioning Anti-Patterns

- AVOID: "AI-powered" without specifics
- AVOID: "Uses GPT-4" (commodity claim)
- AVOID: "Automates everything" (unbelievable)
- AVOID: Hiding how the AI works
- AVOID: Ignoring AI skepticism in the market
- AVOID: Positioning that requires AI literacy to understand

## 10.5 AI Positioning Validation

Test with skeptical buyers:

- [ ] "What happens when the AI is wrong?"
- [ ] "How do I know I can trust this?"
- [ ] "What do humans still do?"
- [ ] "How is this different from [ChatGPT/competitor]?"

If answers don't land, revise positioning.
